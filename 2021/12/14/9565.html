MyBatis的动态SQL

今天和大家聊一下，MyBatis的动态SQL，是什么。动态SQL顾名思义即在某些条件成立或者不成立的情况下，执行的sql发生了变化，进而导致数据库表操作有影响。

1、动态SQL的标签

  • where
  • choose
  • set
  • foreach
  • bind

2、查询操作的应用

  • <choose>类似java中的if else if else 反正只有一个条件成立被执行
<!--根据ID或者用户名查询用户信息-->
<select id="selectByIdOrUserName" resultType="com.wnx.mybatis.model.SysUser">
    SELECT
        id,user_name,user_password,user_email,user_info,head_img,create_time
    FROM sys_user
   <where>
    <choose>
        <when test="id!=null">
            AND id = #{id}
        </when>
        <when test="userName!=null and userName !=''">
            AND user_name = #{userName}
        </when>
        <otherwise>
            AND 1 = 2
        </otherwise>
    </choose>
   </where>

</select>
  • <if> java中的单判断！成立就执行，都成立就一起执行，没有成立的就查询全部
<!--根据用户名或者用户邮箱查询用户信息-->
<select id="selectByUsernameLikeAndUserEmail" resultType="com.wnx.mybatis.model.SysUser">
    SELECT
        id,user_name,user_password,user_email,user_info,head_img,create_time
    FROM sys_user
    <where>
    <if test="userName!=null and userName != ''">
        AND user_name LIKE concat('%',#{userName},'%')
    </if>
    <if test="userEmail!=null and userEmail != ''">
        AND user_email = #{userEmail}
    </if>
    </where>
</select>
  • <band> 经过bind标签来做计算，引用计算新值。比如上面的satement修改，要知道不是所有数据库都是Mysql,Orancle数据库的concat方法之只能连接两个字符串，上面的方式就不能查询了，而bind巧妙的解决了问题。
<!--根据用户名或者用户邮箱查询用户信息-->
<select id="selectByUsernameLikeAndUserEmail2" resultType="com.wnx.mybatis.model.SysUser">
    SELECT
    id,user_name,user_password,user_email,user_info,head_img,create_time
    FROM sys_user
    <where>
        <if test="userName!=null and userName != ''">
            <bind name="userName" value="'%'+ #{userName} + '%'"/>
            AND user_name LIKE #{userName}
        </if>
        <if test="userEmail!=null and userEmail != ''">
            AND user_email = #{userEmail}
        </if>
    </where>
</select>
  • <foreach>标签，可以遍历集合，数组，map.这里说集合和数组，在接口方法如果不用@param指定他的名字的话，集合默认名字list,数组默认名字array,可见下发collection 字段的填入值
<!--根据ID集合查询用户信息-->
<select id="selectByIds" resultType="com.wnx.mybatis.model.SysUser">
    SELECT
        id,user_name,user_password,user_email,user_info,head_img,create_time
    FROM sys_user
    WHERE id IN
    <foreach collection="list" open="(" separator="," close=")" item="id" index="i">
        #{id}
    </foreach>
 </select>
<!--根据ID集合查询用户信息-->
<select id="selectByIds2" resultType="com.wnx.mybatis.model.SysUser">
    SELECT
    id,user_name,user_password,user_email,user_info,head_img,create_time
    FROM sys_user
    WHERE id IN
    <foreach collection="array" open="(" separator="," close=")" item="id" index="i">
        #{id}
    </foreach>
</select>
<!--批量插入用户信息-->
<insert id="insertList" useGeneratedKeys="true" keyProperty="id" keyColumn="id">
    INSERT INTO sys_user(
        user_name,
        user_password,
        user_email,
        user_info,
        head_img,
        create_time
    )
    VALUES
   <foreach collection="list" index="i" item="sysUser"  separator=",">
       (
       #{sysUser.userName},
       #{sysUser.userPassword},
       #{sysUser.userEmail},
       #{sysUser.userInfo},
       #{sysUser.headImg,jdbcType=BLOB},
       #{sysUser.createTime,jdbcType=TIMESTAMP}
       )
   </foreach>
</insert>

4、修改操作的应用

  • 用了<set>标签你就不用考虑sql会出错的问题了。他智能化的帮我去掉最后一个修改字段的逗号.同样的上方的查询，<where>标签也是巧妙避开了当所有条件不成立的时候 where 没有东西了造成sql错误！还有一种很土的办法where 1=1,就和不用<set>标签，在最后搞一个id=#{id}一样，太二了！。
UPDATE sys_user
      <set>
          <if test="userName != null and userName != ''">
              user_name = #{userName},
          </if>
          <if test="userPassword != null and userPassword != ''">
              user_password = #{userPassword},
          </if>
          <if test="userEmail != null and userEmail != ''">
              user_email = #{userEmail},
          </if>
          <if test="userInfo != null and userInfo != ''">
              user_info = #{userInfo},
          </if>
          <if test="headImg != null and headImg != ''">
              head_img = #{headImg,jdbcType=BLOB},
          </if>
          <if test="crateTime != null and crateTime != ''">
              create_time = #{crateTime,jdbcType=TIMESTAMP}
          </if>
      </set>
     WHERE id = #{id}
  • 如果是map，没有在接口方法上加@param注解，那么默认名字叫_parameter ,其实不管是集合啊，数组啊，还是map，底层都在搞到一个map中去了！
<!--通过Map更新-->
<update id="updateByMap">
    UPDATE sys_user
    SET
        <foreach collection="_parameter" item="val" index="key" separator=",">
            ${key} = #{val},
        </foreach>
    WHERE id = #{id}
</update>

5、说一说在if标签的test中你写的东西吧

不管你是在${}还是在bind标签中value属性，还是在各种条件判断表达式子中见到，MyBatis说这些表达式子是OGNL表达式。我认为常用的列举出下面6种！

  • e1 or e2
  • e1 and e2
  • e1 == e2 e1 eq e2
  • e1 != e2 或者 e1 neq e2
  • e1 lt e2 小于
  • e1 gt e2 大于
  • e1 lte e2 e1 gte e2 小于等于 大于等于
  • e1 + e2 e1 - e2 e1 * e2 e1/e2 e1%e2
  • !e not e 取反
  • e.property 对象.属性
  • e1[e2] 按照索引取值（List、Arrray ,Map） 另外Map 也可以 map.key

这个也是常见，集合不能为空格

<if test="list!=null and list.size()>0">
<!--其他-->
</if>

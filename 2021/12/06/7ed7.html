MyBatis上的@Param方法说明

一般我们使用MyBatis进行写接口方法的时候，遇到的方法参数有这么几种情况，要么是一个基础数据类型的，比如说根据ID删除用户信息，要么是一个POJO，比如说新增用户信息，修改用户信息。要么是Map,还有集合，数组。现在呢，这里我们来说一下啊，两个基本数据类型的情况！

1、不使用注解会直接报错

  • 接口方法
/**
  * 根据用户ID和角色的enabled状态获取用户的角色
  * @param userId
  * @param enabled
  * @return
  */
 List<SysRole> selectRolesByUserIdAndRoleEnabled(Long userId,Integer enabled);
  • 接口mapper.xml实现
<!--根据用户ID和角色的enabled状态获取用户的角色-->
<select id="selectRolesByUserIdAndRoleEnabled" resultType="com.wnx.mybatis.model.SysRole">
    SELECT
        r.id,
        r.role_name,
        r.enabled,
        r.create_by,
        r.create_time
    FROM sys_user u
             INNER JOIN sys_user_role ur ON u.id = ur.user_id
             INNER JOIN sys_role r ON ur.role_id = r.id
    WHERE u.id = #{userId} AND r.enabled = #{enabled}
</select>
org.apache.ibatis.exceptions.PersistenceException: 
### Error querying database.  Cause: org.apache.ibatis.binding.BindingException: Parameter 'userId' not found. Available parameters are [0, 1, param1, param2]
### Cause: org.apache.ibatis.binding.BindingException: Parameter 'userId' not found. Available parameters are [0, 1, param1, param2]
  • MyBatis只认识#{0}，#{1}，#{param1},#{param2}这种形式的参数。

修改mapper.xml，采用如下参数名，发现MyBatis可执行该方法了！

WHERE u.id = #{0} AND r.enabled = #{1} 
WHERE u.id = #{param1} AND r.enabled = #{param2}

2、用了注解可以用方法名上定义的形参名了

  • 修改接口加上注解
/**
   * 根据用户ID和角色的enabled状态获取用户的角色
   * @param userId
   * @param enabled
   * @return
   */
  List<SysRole> selectRolesByUserIdAndRoleEnabled(@Param("userId")Long userId,@Param("enabled")Integer enabled);
  • 之前的报错一去不回头了
<!--根据用户ID和角色的enabled状态获取用户的角色-->
<select id="selectRolesByUserIdAndRoleEnabled" resultType="com.wnx.mybatis.model.SysRole">
    SELECT
        r.id,
        r.role_name,
        r.enabled,
        r.create_by,
        r.create_time
    FROM sys_user u
             INNER JOIN sys_user_role ur ON u.id = ur.user_id
             INNER JOIN sys_role r ON ur.role_id = r.id
    WHERE u.id = #{userId} AND r.enabled = #{enabled}
</select>

3、那为啥一个参数的时候，MyBatis不报错呢？

MyBatis在只有一个参数的时候，他不关心这个参数的名字叫啥？直接拿过来方法的值，set进行他的预编译语句。执行！

4、假如是两个POJO作为参数呢？

需要进行SQL查询的字段在SysUser,SysRole对象里面是他们的属性。所以通过.操作符便可以操作到他的属性了！

/**
   * 根据用户ID和角色的enabled状态获取用户的角色
   * @param sysUser
   * @param sysRole
   * @return
   */
  List<SysRole> selectRolesByUserAndRole(@Param("user")SysUser sysUser,@Param("role")SysRole sysRole);
<!--根据用户ID和角色的enabled状态获取用户的角色-->
<select id="selectRolesByUserAndRole" resultType="com.wnx.mybatis.model.SysRole">
    SELECT
        r.id,
        r.role_name,
        r.enabled,
        r.create_by,
        r.create_time
    FROM sys_user u
             INNER JOIN sys_user_role ur ON u.id = ur.user_id
             INNER JOIN sys_role r ON ur.role_id = r.id
    WHERE u.id = #{user.userId} AND r.enabled = #{user.enabled}
</select>

5、总结

用了@Param注解，让自己开发的SQL变得更加清晰，便于他人阅读吧！
